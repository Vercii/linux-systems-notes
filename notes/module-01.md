# Module 1 - Linux
## Linux is a Kernel
**Linux** can refer to either:
- The **Linux kernel** — the core component that manages hardware and system operations.
- The full **GNU/Linux operating system** — a combination of the Linux kernel and GNU software/tools.

Linux originated from the influence of **UNIX**, an operating system developed at AT&T Bell Labs in the 1970s.
- UNIX became popular due to its portability, modular design, and use of the C programming language.
- Many UNIX variants later emerged, though only systems certified by the Open Group can officially use the UNIX name.

In 1991, Linus Torvalds created the Linux kernel as a hobby project while studying at the University of Helsinki.
- He was inspired by MINIX, a UNIX-like educational operating system, but wanted a more complete and flexible system.
- Linux eventually became one of the most widely used operating systems, especially for servers and Internet infrastructure.

Linux is considered **UNIX-like** because:
- It follows many UNIX design principles and standards.
- However, it is not officially UNIX-certified.

The **GNU Project**, founded in 1983 by Richard Stallman, developed free and open-source tools such as:
- Compilers
- Editors
- Command-line utilities

GNU tools were combined with the Linux kernel to create the complete **GNU/Linux operating system** used today.

## Linux is Open Source
Traditionally, most software was released under a **closed-source license**:
- Users could run the software but could not access or modify its source code.
- Many licenses also prohibited reverse engineering the program.
  
The rise of Linux closely followed the growth of the **open source software movement**.
- Open source promotes the idea that users should be able to:
    - Access source code
    - Modify software
    - Share improvements with others

Linus Torvalds made the Linux source code freely available, allowing developers worldwide to contribute to the project.
- Early users could directly influence Linux’s development.
- Developers shared modifications and improvements openly, speeding up innovation and reducing repeated mistakes found in other operating systems.

This collaborative development model became one of the key reasons for Linux’s rapid growth and success.

## Linux has Distributions
When people say their computer runs **Linux**, they usually mean a complete **Linux distribution (distro)** rather than just the Linux kernel.
A distribution combines:
- The Linux kernel
- GNU tools and utilities
- User applications such as web browsers and email clients

Linux distributions also includes:
- Installation tools for setting up storage and the operating system
- System management utilities
- A **package manager** used to install, update, and remove software

Different Linux distributions are designed for different purposes, including:
- Servers
- Desktop computers
- Specialized fields such as electronics design or statistical computing

Most major Linux distributions trace their origins to three main families:
- Red Hat
- Debian
- Slackware

One of the biggest differences between Red Hat-based and Debian-based systems is the **package manager**, though they also differ in:
- File system organization
- System configuration methods
- Development philosophies and goals

## Linux Embraces the CLI
There are two main ways to interact with an operating system:
- **Graphical User Interface (GUI)**
- **Command Line Interface (CLI)**

**Graphical User Interface (GUI):**
- Uses windows, icons, menus, and pointers.
- Applications appear in resizable/movable windows.
- Common GUI applications include:
    - Web browsers
    - Email clients
    - Graphics and photo editing tools
- Designed to be user-friendly and visually intuitive.

**Command Line Interface (CLI):**
- Text-based interface controlled using keyboard input.
- Users type commands to perform actions instead of clicking icons.
- Emphasizes direct control: the user explicitly instructs the computer what to do.

Key idea:
- In a GUI, the system guides the user through visual elements.
- In a CLI, the user directly instructs the system through commands.

Most operating systems (like Windows and macOS) provide both interfaces but emphasize the GUI for general users.

The Linux community strongly embraces the CLI because it is:
- Powerful
- Fast
- Capable of performing complex tasks efficiently with single commands
