# Module 4 – Open Source Software & Licensing

## Source Code and Compilation
- Software is written as **source code** (human-readable instructions).
- A **compiler** converts source code into executable machine code.
- Some languages are **interpreted** and run line-by-line:
    - Python
    - Perl
    - Bash

---

## Closed Source vs Open Source

### Closed Source Software
- Only executable files are provided.
- Source code is hidden.
- Controlled by restrictive licenses/EULAs.
- Reverse engineering and redistribution are usually prohibited.

Example:
- Microsoft Windows

### Open Source Software
- Source code is publicly available.
- Users may inspect, modify, and redistribute software under license terms.
- Encourages:
    - Collaboration
    - Transparency
    - Security auditing

Example:
-  Linux

---

## Benefits of Open Source
- Community development improves:
    - Security
    - Stability
    - Compatibility
    - Innovation
- Public code allows vulnerabilities to be identified faster.
- Internet collaboration enables rapid global contributions.

---

## Linux and UNIX Influence
- Linux adopted many UNIX design principles.

- UNIX milestones:
    - Rewritten in **C** (1973)
    - BSD UNIX introduced **TCP/IP** (1984)
- Linux became successful because it was:
    - Open
    - Portable
    - Standards-compatible
    - Community-driven

---

## Standards and Compatibility
- Standardized **APIs** improve software portability.
- Important standards/groups:
    - **IEEE**
    - **POSIX**
    - **X/Open**

# Software Distribution Components
- **Ownership** – who owns the software
- **Money Transfer** – how payment works
- **Licensing** – rules for usage/modification/distribution

---

## GNU GPL and FOSS

### GNU GPLv2 Principles
- Source code must remain available.
- Users may modify and redistribute software.
- Modified versions must remain under the same license.
- Fees may only cover distribution costs.

### FOSS
- **FOSS** = Free and Open Source Software
- “Free” means **freedom**, not price.

---

# Free Software Foundation (FSF)
- Founded in 1985 by Richard Stallman
- Promotes software freedom:
    - Use
    - Study
    - Modify
    - Share

## Copyleft
- Modified versions must also remain open/free.
- Preserves user freedoms in future versions.

## GNU Licenses
- GPLv2
- GPLv3
- LGPLv2
- LGPLv3

## FSF Positions
- Opposes:
    - Software patents
    - DRM restrictions

## GPLv3 and Tivoization
- GPLv3 prevents hardware makers from blocking modified software.
- Linus Torvalds kept Linux under GPLv2 instead of GPLv3.

---

## Open Source Initiative (OSI)
- Founded in 1998 by:
    - Bruce Perens
    - Eric S. Raymond

- Promotes a practical approach to open source.
- Approves licenses using the **Open Source Definition**.
- Does not create licenses itself.

---

# Permissive Licenses

## BSD & MIT Licenses
- Allow modification and redistribution.
- Require attribution.
- Modified versions may become proprietary.

Examples:
- BSD 2-Clause
- BSD 3-Clause
- MIT License

---

## FOSS vs FLOSS
- **FOSS** = Free and Open Source Software
- **FLOSS** = Free/Libre/Open Source Software
- “Libre” emphasizes freedom rather than cost.

---

# Creative Commons (CC)

## Purpose
- Designed for non-software content:
  - Documents
  - Art
  - Music
  - Media

## Common CC Conditions
- **BY** – Attribution required
- **SA** – Same license required for derivatives
- **NC** – Non-commercial only
- **ND** – No modifications allowed

## Main CC Licenses
- **CC BY** – credit required
- **CC BY-SA** – share-alike required
- **CC BY-ND** – no derivatives
- **CC BY-NC** – non-commercial
- **CC BY-NC-SA** – NC + share-alike
- **CC BY-NC-ND** – NC + no derivatives
- **CC0** – public-domain equivalent
